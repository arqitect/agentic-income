# Offer Packages And Pricing

Pricing is intentionally practical for a student operator building local proof. The first goal is a paid beta and repeatable fulfillment, not pretending to be a large agency.

## Pre-Revenue Selling Rule

Before the first paid client, sell only:

- `Beta Setup`
- `Local Quote System`

Keep the other packages documented, but do not lead with them cold unless a prospect clearly asks for that scope or arrives through a warm introduction.

## Package Details

## Beta Setup

### Deliverables

- quick audit
- one improved contact or quote path
- simple CRM sheet
- one manual or semi-automated follow-up reminder flow
- review request templates

### What Client Must Provide

- current website access or contact form details
- business phone and email details
- logo and service list
- one main contact person

### What Eric Must Do

- audit the existing intake flow
- simplify the quote path
- set up the tracker
- test the new flow manually
- provide a short handoff

### Risks

- client has no website access
- client changes scope mid-project
- client expects ad generation, not system cleanup

## Local Quote System

### Deliverables

- revised service page or lightweight landing page
- quote form with better qualifying fields
- lead tracker with stages
- follow-up reminder workflow
- basic analytics or reporting view

### What Client Must Provide

- service area list
- phone and email
- testimonials or reviews
- photo assets if available

### What Eric Must Do

- structure the intake logic
- build the tracker
- write or revise the page copy
- test mobile flow
- train the client

### Risks

- client wants full site rebuild
- photo or testimonial assets are missing
- client response habits do not improve after setup

## Review + Follow-Up System

### Deliverables

- review request email/SMS templates
- job-complete follow-up cadence
- lead resurrection checklist
- monthly review tracker

### What Client Must Provide

- review links
- current customer communication method
- staff member or owner responsible for sending

### What Eric Must Do

- map current follow-up behavior
- write message templates
- set the reminder workflow
- test the process on sample jobs

### Risks

- client expects fully autonomous texting without approvals
- client does not have clear customer consent practices for SMS
- no one owns execution internally

## Property Manager Ops System

### Deliverables

- maintenance request intake
- priority triage fields
- vendor tracking sheet or base
- resident or owner update templates
- simple dashboard

### What Client Must Provide

- current process map
- categories for maintenance requests
- main escalation contacts
- preferred approval flow

### What Eric Must Do

- document the workflow
- build the intake and tracker
- create role-based views
- test with example tickets

### Risks

- too many edge cases in the first version
- sensitive tenant data handling
- client wants a full property-management platform replacement

## Local SEO + Intake System

### Deliverables

- local service pages
- location/service keyword plan
- intake form
- GBP recommendation document
- review workflow support

### What Client Must Provide

- target cities and services
- photos
- service descriptions
- current GBP link

### What Eric Must Do

- draft location-aware content
- build or recommend page structure
- tie the page to a real intake path
- provide GBP cleanup recommendations

### Risks

- client expects ranking guarantees
- pages are published without verification of claims

## Pricing Notes

- First two clients can receive beta pricing in exchange for fast feedback and permission to reference anonymized results.
- Raise pricing after one clean case study or testimonial.
- Do not discount custom work below the cost of Eric's time just to "get experience."
- If a prospect needs more than one page, more than one workflow, or major website work, move it out of beta pricing.
