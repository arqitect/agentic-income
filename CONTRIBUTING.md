# Contributing

Thanks for contributing.

## Scope

This repository is an operations playbook. Prefer practical, testable changes over abstract additions.

## Contribution rules

- Keep claims evidence-based; include sources for market/legal assertions.
- Do not add workflows that bypass human approval for outreach.
- Keep templates concise and specific to local service-business outcomes.
- When changing schemas (`22_data/` or `27_config/`), update all dependent prompts/docs.

## Pull request checklist

- Explain why the change is needed.
- List files changed and impact.
- If changing schemas, include migration notes.
- Confirm no guarantee language or prohibited outreach automation was introduced.
