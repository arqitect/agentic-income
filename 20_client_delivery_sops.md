# Client Delivery SOPs

## Global Approval Rule

No live page, form, workflow, review request, or client-facing copy goes live without human approval from both Eric and the client contact.

## Landing Page SOP

### Process

1. confirm service and target city
2. gather proof points, photos, reviews, and CTA
3. draft page structure
4. write plain, local copy
5. test mobile flow
6. connect CTA to form or contact path

### Acceptance Criteria

- clear service offer
- clear CTA
- mobile readable
- city/service claim verified by client
- client approves the live copy before publication

## Quote Form SOP

### Process

1. identify required qualification fields
2. remove unnecessary friction
3. define where submissions go
4. test success message and notification path
5. confirm lead enters tracker correctly

### Acceptance Criteria

- form works on desktop and mobile
- required fields are useful
- leads route to the correct place
- client approves any live-field changes before publication

## CRM Sheet SOP

### Process

1. create lead stages
2. define owners and next-step field
3. add date fields for contact, proposal, close, follow-up
4. test with one sample lead

### Acceptance Criteria

- team can understand stages
- sample lead moves through correctly
- no mandatory field blocks normal use

## Review Request Workflow SOP

### Process

1. get correct review links
2. choose trigger moment
3. write SMS/email templates
4. set reminder logic
5. confirm opt-out handling

### Acceptance Criteria

- review links work
- message copy is approved
- reminder timing is documented
- if SMS is used, consent handling is documented first

## Follow-Up Workflow SOP

### Process

1. define the lead stages that need follow-up
2. define timing
3. write draft message templates
4. assign human owner
5. log every follow-up

### Acceptance Criteria

- follow-up timing is clear
- owner knows what to do
- no automated sending occurs without approval

## Local SEO Page Draft SOP

### Process

1. choose service + city pair
2. verify the client actually serves that area
3. draft page with local relevance
4. avoid copied or generic filler
5. connect the page to intake

### Acceptance Criteria

- page is specific to the service and geography
- claims are verified by client
- CTA is present
- page is not published until client signs off on factual accuracy

## Google Business Profile Recommendation Doc SOP

### Process

1. review visible profile basics
2. list missing or inconsistent items
3. identify review-response opportunities
4. draft recommendations only

### Acceptance Criteria

- recommendations are concrete
- no ranking guarantees
- business facts are verified

## Basic Analytics / Reporting SOP

### Process

1. define which metrics matter
2. pull from tracker, not guesswork
3. summarize leads, replies, proposals, wins
4. include next actions

### Acceptance Criteria

- client can understand the report in 3 minutes
- numbers match source records
- next actions are practical
